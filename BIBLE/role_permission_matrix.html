Пользователь может иметь несколько ролей одновременно. Типичные рабочие комбинации:
Обычный студент
student
Самый базовый случай. Только записывается на курсы, смотрит уроки, получает сертификаты.
Студент-преподаватель
studentlecturer
Учится сам и одновременно ведёт свои курсы. Типичный сценарий для середняка.
Фрилансер
executor
Берёт проекты от клиентов, работает по контрактам, выставляет услуги в маркетплейсе.
Фрилансер с субподрядом
executorclient
Берёт крупный проект как executor и сам нанимает других исполнителей как client.
Студия / агентство
clientlecturer
Заказывает работу у фрилансеров и параллельно ведёт обучающие курсы для своей аудитории.
Универсальный участник
studentlecturerexecutor
Учится, преподаёт и берёт проекты. Нет ограничений на комбинирование пользовательских ролей.
Контент-модератор
moderator
Только модерация. Не создаёт контракты, не публикует курсы — только проверяет и банит.
Операционный менеджер
manager
Включает все права moderator. Создаёт контракты, выдаёт сертификаты, видит всю аналитику.
Суперадмин
admin
Включает все права. Email redgriffinanimatestudio@gmail.com получает auto-grant без записи в roles[].
⚠️ Невалидные комбинации
managermoderator
Manager уже включает все права moderator — дублировать бессмысленно. Аналогично admin + manager.
